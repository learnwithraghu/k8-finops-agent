### Block 1 ‚Äî Why `kustomization.yaml` exists (~60 sec)

> **Do:** Open `kustomization.yaml` at the root. Scroll to the `resources:` list.

This is the entry point. Six folders listed here. When you run `kubectl apply -k` on this directory, Kustomize reads this file first, pulls in every listed folder, and applies the result.

Why not one giant YAML file? Because each team owns a folder. Kustomize lets you compose many small manifests into one deploy without Helm or a templating engine. It is built into kubectl.

One file tells kubectl *what* to deploy. The folders tell it *how each piece is shaped*.

> **Do:** Name the five services as you scroll ‚Äî booking, flight search, inventory, payment, analytics. Pause on `problem-resources` at the bottom.

We will come back to that one.

> **Do:** Open `flight-search-service/kustomization.yaml`.

Two things to notice. First, `namespace: flight-search` ‚Äî Kustomize stamps every resource in this folder into that namespace. Second, the four files listed under `resources:`. Deployment, Service, ConfigMap, PVC. That is the standard microservice bundle in Kubernetes.

> **Do:** Glance at `service.yaml`, `configmap.yaml`, and `pvc.yaml` in the same folder ‚Äî point at them, do not open.

- **ConfigMap** ‚Äî configuration without rebuilding the image
- **PVC** ‚Äî a storage request that survives pod restarts

We will not open all five service copies. Once you see the pattern in one folder, you have seen it everywhere.

### Block 3 ‚Äî The workload itself (~60 sec)

> **Do:** Open `flight-search-service/deployment.yaml`. Skip line-by-line ‚Äî jump to three spots only.

Three things worth calling out in this file.

First, **`image: nginx:alpine`**. There is no airline application source code in this repo. A tiny web server stands in for the real microservice. We are teaching Kubernetes operations, not airline business logic.

Second, **`containerPort: 80`** ‚Äî the port the container listens on.

Third, **`resources` requests and limits** ‚Äî what the scheduler reserves and what FinOps will eventually care about.

> **Do:** Scroll to the labels section. Point at the comment about missing tags.

Notice `environment: prod` is there, but comments call out missing `owner` and `cost-center`. File that away ‚Äî Section 03 hunts for exactly this kind of gap.

The Deployment is desired state. Everything else in the folder supports it.
