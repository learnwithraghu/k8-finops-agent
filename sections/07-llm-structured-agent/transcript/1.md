### Opening (~20 sec)

Welcome to Section 07. In Section 06 you ran a LangChain agent that audited cluster labels and wrote a free-text report. It flagged missing owner and cost-center labels — but only because we told it to in the prompt.

Today we add a policy file. Same agent. Same MCP tools. The rules live in `config/tagging-rules.yaml` and the script reads them at runtime.

### What Section 06 already does (~25 sec)

`label_auditor.py` is thin: a prompt, a call to `run_agent`, print the answer. The LLM chooses MCP tools, gathers label data, and writes a human-readable summary.

That is the right shape. We keep it.

> **Visual:** Highlight the three lines in `label_auditor.py` — PROMPT, run_agent, print.

### What Section 07 adds (~30 sec)

One thing: policy from file.

`structured_auditor.py` loads `tagging-rules.yaml` and passes it to `run_agent` as a separate system message. Required tags, label mappings, excluded namespaces — the standard the agent checks against.

The output is still plain English on screen. No ticketing. No JSON schema. Just a better-informed audit.

> **Visual:** Flow diagram — instruction prompt + rules file → MCP agent → plain-English answer on screen.
