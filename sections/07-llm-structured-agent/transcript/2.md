### The policy file (~90 sec)

`tagging-rules.yaml` lives in `config/` next to the code. The script reads it with `load_tagging_rules()` — we do not paste YAML into the prompt constant.

**`required_tags`** — labels every application resource must have: owner, environment, cost-center, application, tier.

**`label_mappings`** — alternative Kubernetes label keys the agent should check. A resource might use `app.kubernetes.io/owner` instead of `owner`.

**`excluded_namespaces`** — system namespaces we skip: kube-system, kube-public, local-path-storage.

**`cost_centers`, `environments`, `tiers`** — allowed values the agent can reference when explaining gaps.

> **Visual:** Scroll through the YAML, highlight each section as named.

> **Say:** "This file is policy, not code. Change the rules, re-run the auditor — no prompt edits needed."

---

### How the script uses it (~45 sec)

In `structured_auditor.py`, `load_tagging_rules()` returns the file as text. `main` passes it to `run_agent` as `tagging_rules=...`.

Inside `mcp_client`, that becomes a system message: "Apply the tagging rules below…" followed by the YAML. The instruction prompt stays separate — what to do, not what the policy is.

> **Visual:** Show `load_tagging_rules()` and the `run_agent` call in `structured_auditor.py`. Then show the system message in `mcp_client.py`.

> **Say:** "Rules from file. Instructions in the prompt. Agent calls MCP tools. Plain English out."
