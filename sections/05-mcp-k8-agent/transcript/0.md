# Video: MCP in 90 Seconds — What It Is and Why Agents Need It

**Time Budget:** 1:30

### Opening

An agent that only chats is not useful. It needs to *do things* — query a database, file a ticket, read a file. Today every integration is bespoke: a custom Python script here, a one-off API wrapper there. That does not scale.

---

### What is MCP

**MCP** — the Model Context Protocol — is an open standard for connecting AI clients to external capabilities.

Think client and server. The client — an agent, an IDE, a script — sends JSON-RPC messages. The server advertises what it offers: **tools** to invoke actions, **resources** to read data, and **prompts** for reusable workflows.

Locally the transport is usually stdio — the client spawns the server as a subprocess. Remotely it is HTTP (Streamable HTTP or SSE). Same protocol either way.

This is not just another REST wrapper. It is a **protocol** any compliant client can speak — Cursor, VS Code, Python, curl.

---

### Why agents need it

Without MCP, every new data source means a new integration you own forever. With MCP, the server implements the protocol once; every client can discover and call its tools through the same handshake.

That handshake matters. The client sends `initialize`. The server replies with its capabilities and tool list. Then the client calls `tools/call` with a tool name and arguments. Structured request in, structured result out.

One standard seam between intelligence and action — instead of N custom adapters.
