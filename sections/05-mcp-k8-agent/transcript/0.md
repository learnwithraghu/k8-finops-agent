# Video: MCP in 90 Seconds — What It Is and Why Agents Need It

**Time Budget:** 1:30

### Opening

An agent that only chats is not useful. It needs to *do things* — read a cluster, check a deployment, pull logs. Today every integration is bespoke: a custom Python script here, a one-off API wrapper there. That does not scale.

---

### What is MCP

**MCP** — the Model Context Protocol — is an open standard for connecting clients to tools and resources.

A client sends JSON-RPC messages. The server advertises what it can do: tools like `kubectl_get`, `kubectl_describe`, `kubectl_logs`. Locally the transport is stdio. Remotely it is HTTP or SSE.

This is not a REST wrapper around kubectl. It is a **protocol** any client can speak — curl, Python, VS Code, a future agent.

---

### MCP in Kubernetes

For Kubernetes there is a prebuilt image: `mcp/kubernetes`. It wraps kubectl as MCP tools. Mount your kubeconfig, and the server gets read-only cluster access — the same credentials kubectl already uses.

In Section 04 we designed a Gather stage that needs a maintainable read path. MCP is that interface: one standard seam instead of a custom collector you own forever.
