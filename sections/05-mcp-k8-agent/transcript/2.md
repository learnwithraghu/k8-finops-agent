# Video: Demo — Prove the Chain with curl (Health, Handshake, and a Cluster Read)

**Time Budget:** 3–4 min

> **Legend:** Lines in blockquotes are presenter cues — actions and timing. Do not read them aloud. Everything else is your script.

---

### Opening (~20 sec)

Supergateway is running in another terminal. Now we prove the chain with raw HTTP — no Python, no SDK, no client library. If curl works, any MCP client works. That is the bar we set before building agents on top.

> **Do:** Open a **new** terminal. Supergateway stays running in the other one.

---

### Step 1 — Health probe (~30 sec)

Start with the simplest check: is the HTTP layer up?

> **Do:** Run `curl -s http://localhost:8000/healthz`.

Expected: `ok`. Health checks are table stakes for any service. This confirms Supergateway is listening before we try MCP protocol messages.

---

### Step 2 — MCP initialize handshake (~60 sec)

Now the MCP handshake. Every client — curl, Python, VS Code — sends `initialize` first. The server responds with its name and the tools it supports.

> **Do:** Run the POST to `/message` with the `initialize` JSON body.

Expected: a JSON response with server name **"kubernetes"** and a list of supported tools — `kubectl_get`, `kubectl_describe`, and others.

This is not kubectl yet. This is the protocol layer confirming what the server can do. The client and server agree on capabilities before any tool is called.

---

### Step 3 — Real cluster read (~60 sec)

Now a real cluster read. We call the `kubectl_get` tool to list namespaces.

> **Do:** Run the POST to `/message` with the `tools/call` body for `kubectl_get` / namespaces.

Expected: a clean list — `booking-api`, `flight-search`, `inventory`, `payment`, and the rest of the SkyLine Air footprint.

We just read the cluster through MCP using nothing but curl. No Python. No SDK. No kubeconfig on *this* terminal. The MCP server inside Docker handled authentication and execution.

That is the Gather seam Section 06 will automate.

---

### Step 4 — Cleanup (~20 sec)

When you are done validating, stop the gateway.

> **Do:** Switch to the Supergateway terminal and press `Ctrl-C`.

The MCP server and Supergateway shut down together. Local demo state stays predictable.

---

### Interactive — Architecture builder (~30 sec)

> **Do:** Open `architecture_builder/index.html` in your browser.

Stack the gateway layers bottom-up — kubeconfig, Docker MCP server, Supergateway, curl client. Complete the curl proof ladder. Use **Need a hint?** if stuck, then press **Test Gateway** to validate.

This is the same chain you just ran in the terminal, visualized.

---

### Close (~20 sec)

MCP endpoint validated. Health, handshake, real cluster read — all through HTTP on port 8000.

Next up: Section 06 builds a Python agent that calls this same endpoint and assembles a cluster snapshot. The seam is proven. Now we automate it.
